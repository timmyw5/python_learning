## 🌉 从Excel出发

你在Excel里有没有给单元格取过名字？比如把A1命名为"总价"，然后公式里写 `=总价 * 0.9`？

Python的变量就是这个意思——**给一个值起个名字，方便后面用**。

```
Excel:   总价 = A1的值
Python:  total = 100
```

就这么简单。`total` 是名字，`100` 是值，`=` 是"把右边的值存到左边的名字里"。

---

## 📖 变量的本质：贴标签

变量不是一个"盒子"，更像是一个**标签**。你把标签贴到东西上：

```python
price = 99          # 给99贴了个标签叫price
name = "Timmy"      # 给"Timmy"贴了个标签叫name
is_vip = True       # 给True贴了个标签叫is_vip
```

标签可以**撕下来贴到别的东西上**：

```python
price = 99          # price → 99
price = 199         # price → 199（标签从99撕下来，贴到199上了）
```

现在 `price` 就是199了。旧的99没人管了。

## 👀 认识赋值语句的各种样子

你以后会在代码里看到这些写法，全都是赋值：

```python
# 最基本的：名字 = 值
x = 10
message = "你好世界"

# 用计算结果赋值
total = 50 + 30          # total 是 80
half = total / 2         # half 是 40.0

# 用别的变量赋值
original = 100
copy = original          # copy 也是 100

# 用函数的结果赋值（1.1提到的）
length = len("Python")   # length 是 6

# 用一行给多个变量赋值（偶尔会见到）
a, b, c = 1, 2, 3       # a=1, b=2, c=3
```

> 🔗 **Excel类比**：每一行就像 `A1=10`、`B1=A1+30`。只是Python用名字代替了单元格编号。

## 🏷️ 命名规则：Python的"起名规矩"

Python对名字有一些规矩。**不用背**，但看到的时候要认得：

### ✅ 合法的名字

```python
name = "Timmy"        # 纯字母 ✅
user_name = "Timmy"   # 下划线连接 ✅（最常见的风格）
price2 = 99           # 字母+数字 ✅
_private = "secret"   # 下划线开头 ✅（有特殊含义，以后会讲）
```

### ❌ 不合法的名字

```python
2name = "Timmy"       # ❌ 不能以数字开头
user-name = "Timmy"   # ❌ 不能用减号（Python以为你在做减法）
my name = "Timmy"     # ❌ 不能有空格
class = "高级"         # ❌ 不能用Python保留词（class是Python自己用的）
```

### 🎯 一眼看出好名字 vs 烂名字

```python
# 烂名字 —— 看了不知道是什么
a = 99
b = "深圳"
x = True

# 好名字 —— 一看就懂
price = 99
city = "深圳"
is_member = True
```

> 💡 **AI时代你需要知道的**：当你让AI写代码时，如果它给变量起了像 `a`、`b`、`temp1` 这种烂名字，你应该要求它改成有意义的名字。好名字 = 代码自带说明书。

## 🔍 `=` vs `==`（1.1答疑回顾）

上次你问过这个，这里正式归纳：

```python
# 一个等号 = 赋值（动作：存！）
name = "Timmy"       # 把"Timmy"存到name里

# 两个等号 == 比较（提问：一样吗？）
name == "Timmy"      # True（是的，一样）
name == "Jerry"      # False（不，不一样）
```

你可以这样记：
- `=` → 一个箭头 → **"把右边的给左边"**
- `==` → 天平 → **"两边一样重吗？"**

---

## 📋 总结卡片

| 概念 | 一句话 |
|------|--------|
| 变量 | 给值贴标签，方便后面用 |
| `=` | 赋值：把右边的值存到左边的名字里 |
| `==` | 比较：两边一样吗？返回True/False |
| 命名规则 | 字母/下划线开头，不能有空格和减号，不能用保留词 |
| `snake_case` | Python最常见的命名风格：小写+下划线，如 `user_name` |
| 好名字 | 一看就知道是什么的名字（`price` > `a`） |

> 📌 **不用背规则**。以后在代码里看到 `total_price = base_price * 1.1` 你能认出"这是在算总价"就够了。
